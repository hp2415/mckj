# 业务全景：WeChat AI 双轨制跟单系统 (Phase 15 目前进度)

##  项目愿景与架构定位
本项目旨在不侵入修改微信底层代码逻辑的前提下，利用**桌面端吸附侧边栏助手**、**高效异步 FastApi 微服务**和 **AI 全能推理模型（Dify）**相结合，赋能传统业务员实现高效的精准客户维系与商品下发分销。

---

##  核心三端联动架构解析

### 1. 桌面客户端 (PySide6 / qasync) —— 敏捷指挥舱
桌面端打磨成了“超轻量化随动跟单模块”，提供毫秒级的业务支持体验：
- **IM级极简交互体系**：深灰色胶囊样式侧边栏导航、半透隐藏滚动条、全组件增加视觉悬浮态反应。
- **黑科技全景赋能**：
  - AI 对话全局监听 `Enter` 秒发与 `Ctrl+Enter` 换行，向主流办公 IM 软件对齐操作习惯。
  - 货源中心支持“无极点击入板”：图片单击提取到系统剪贴板可立刻 `Ctrl+V` 到微信发给客户，商品名称与链接同理可以单独提取。
- **状态探测感应器**：自动心跳侦测机制通过监控后台货源码（接口返回 `status` 和 `last_updated_at`），实现了智能预警上链（同步超时24小时将红色警示），配合专属的 Admin 高级一键即时同步拉取动作。
- **桌面三级穿透缓存策略**：避免后端图片 IO 瓶颈，在用户搜索时通过 L1内存+L2基于工号隔离加密SQLite缓存图片资产。

### 2. FastAPI 中台引擎 (Python 3.11) —— 核心流控制阀
完全分离解耦的异步服务核心枢纽，专注于数据一致性和队列分发。
- **级联绑定的自然数据流转**：通过采用主观/客观多表自然键关联逻辑分离表，即便某个业务员出现人事变动，也可对所有带客关系（含 AI 获取得出的记忆）下发秒级一键转移操作。
- **动态防穿透策略屏障**：系统所有外部接口包含 AI 提供商 API Key 还是对接的 832 商品平台，全系引入 `system_configs` 动态刷新管理。搜索接口通过 `supplier_ids` 的 IN 条件检测实现了前端货品的“实时配置化逻辑下线与重发布”。
- **APScheduler 自动化队列引擎**：后台提供 `boot_sync_832` 和 `daily_sync_832`，处理每日长尾图片的破防盗链代理下载与信息更新入库，并规范留下成功和异常时间戳元信息。

### 3. Dify LLM 智能体 —— 持续上下文洞察机制
- 使用特殊的 `dify_conversation_id` 持久化跟随具体客户与特定员工的关系维度。无论桌面端如何开关或断电，都可以衔接上次的谈话推演逻辑，帮助员工提取出客户关键的非结构化画像资料。
