#  AI 微信助手后端项目全景图 (Backend Overview)

本后端基于 **FastAPI + SQLAlchemy (异步) + SQLAdmin** 打造，具备高性能、全异步、图形化管理的特点。以下是代码目录的详细说明：

---

##  根目录文件 (Root Files)

- **`main.py`**  
   **系统心脏**：整个应用的入口。负责初始化 FastAPI、挂载所有业务路由（API）、配置管理后台（Admin）、注册定时任务、以及开放静态资源目录（Media）。
  
- **`models.py`**  
   **数据蓝图**：定义了所有 MySQL 表结构。包括：
    - `User`: 系统员工/管理员信息
    - `Product`: 采集到的 832 商品缓存
    - `Customer`: 核心客户信息库
    - `ChatMessage`: 微信聊天记录存档
    - `SystemConfig`: 存储动态配置（如店铺 ID、Dify Key）

- **`database.py`**  
   **数据管道**：建立与 MySQL 的异步连接池，提供 `get_db` 依赖注入给 API 使用。

- **`admin_views.py`**  
   **后台界面定制**：专门用来写管理后台展示效果的代码。我们在此实现了后台全中文化、屏蔽密码显示、以及“防呆式”下拉菜单选择器。

- **`migrate.py` / `init_db.py`**  
   **维护工具**：前者用于无损给表增加新字段，后者用于初次运行项目时一键建表。

- **`create_superuser.py`**  
   **钥匙工厂**：用来在系统为空时创建第一个超级管理员账号。

---

##  `api/` (业务接口层)
*这里定义了桌面端（Desktop）与之通信的所有逻辑节点：*

- **`auth.py`**: 处理员工登录、颁发 JWT 令牌。
- **`product.py`**: 提供毫秒级的商品本地搜索，以及“一键手动强制同步”货源的特权接口。
- **`customer.py`**: 管理客户资料、画像及归属关系。

---

##  `core/` (核心逻辑层)
*系统的内部引擎和安全机制：*

- **`tasks.py`**  
   **自动化爬虫引擎**：内置 APScheduler 调度器。负责：
    - 凌晨 3 点自动查房
    - 多店铺并发垂直同步
    - 突破 403 封锁下载原图到本地

- **`security.py`**  
     **安全哨兵**：负责密码的 Bcrypt 加密 Hash、JWT 令牌的生成与校验逻辑。

- **`admin_auth.py`**  
     **后台锁匠**：专门针对 `/admin` 页面实现的独立登录身份验证逻辑。

---

##  其他目录 (Other Dirs)

- **`media/`**  
   **本地 CDN 仓库**：物理存储下载回来的商品图片，由 FastAPI 静态托管。
  
- **`.env`**  
   **机密文件**：存储数据库连接字符串、加密密钥等绝密环境变量。

- **`test_*.py`**  
   **化验室**：用于诊断 832 平台原始数据流或测试 API 连通性的临时脚本。

---

> [!NOTE]
> **开发规范**：
> 1. 修改表结构请去 `models.py`。
> 2. 这里的代码全部为 **Async 异步** 实现，调用数据库时必须配合 `await` 关键字。
